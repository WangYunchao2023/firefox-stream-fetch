# firefox-stream-fetch v3.1 使用说明

## 一句话

patched Firefox 两阶段抓取：Phase 1 打开页面、过 CF/登录 → Phase 2 同 profile 从头重放，StreamDumper 抓解密后的 H.264/HEVC/AV1 + AAC → 自动合流 .mp4。

---

## 快速开始

```bash
cd /home/wangyc/Documents/软件类工作/firefox-stream-fetch/scripts

# 1. 明文直连（最快验证，无 CF、无代理）
./capture.sh "https://www.olevod.com/player/vod/1-82695-1.html"

# 2. 需代理站（YouTube / 国外）— 系统已配置代理时直接跑即可
./capture.sh "https://www.youtube.com/watch?v=_n4SRDYkhqs"

# 3. 需 CF 验证站（yfsp 等）
./capture.sh "https://www.yfsp.tv/watch?v=ztgsSWh5mPZEhhazLjYUG6"
# → Phase 1 你手动点过验证 → 自动切 Phase 2 抓流

# 4. 不传 URL 直接启动（空白页，手工输入地址栏）
./capture.sh
# → Phase 1 打开 about:newtab，你在 Firefox 地址栏手输 URL、过验证、播放
# → 检测到标题变化 → 自动切 Phase 2 从头抓取

# 5. 保留中间文件 / 自定义输出目录
./capture.sh "https://..." --keep-h264 --output /data/videos

# 6. 自动连集（每集独立 mp4：host-01.mp4, host-02.mp4...）
./capture.sh "https://www.yfsp.tv/playlist/xxx" --auto-next
```

---

## 两阶段流程

```
Phase 1（用户交互，sandbox 开，真实 UA）
  └─ 启动 Firefox → 打开 URL → 手动过 CF/登录 → 标题变视频标题
     → 自动 SIGTERM 保存 session/cookie → 检测 cf_clearance → Phase 2

Phase 2（自动抓取，sandbox 关）
  └─ 同 profile 重启 Firefox → 从头加载 → StreamDumper 写 .h264/.aac
     → lib-monitor 每 5s BiDi 轮询（ended/stalled/paused/next_episode）
     → 结束触发 → ffmpeg -c copy 合流 .mp4 → 清理中间文件
```

## 输出

默认 `~/Videos/firefox抓取/`

| 文件 | 说明 |
|---|---|
| `host-01.mp4` / `host-timestamp.mp4` | 最终产物（可播放） |
| `host-xxx.log` | Firefox + monitor 日志 |
| `host-xxx.sidecar.json` | 进度、关键帧、切集 URL、中断历史 |

* `--auto-next` 开启时：`host-01.mp4`, `host-02.mp4`... 顺序命名  
* 单集/关闭时：`host-timestamp.mp4`

## 参数速查

| 参数 | 作用 | 默认 |
|---|---|---|
| `--output DIR` | 保存目录 | `~/Videos/firefox抓取/` |
| `--auto-next` | **自动连集**（每集独立 mp4） | 否 |
| `--keep-h264` | 保留 .h264/.aac | 合流后删 |
| `--skip-phase1` | 跳过 Phase 1（profile 已有 cookie） | 否 |
| `--profile PATH` | 自定义 profile | `/tmp/firefox-stream-{host}` |

## 代理（自动读系统设置，无需额外配置）

**默认已在 prefs 里启用**：`network.proxy.type=5`（使用系统代理设置）。

| 场景 | 需要额外操作吗？ |
|---|---|
| 系统已配置代理（设置 → 网络 → 网络代理） | **不需要**，Firefox 自动走系统代理 |
| 想临时换个代理端口 / 临时走直连 | 可 `export http_proxy=...` 覆盖，或改 prefs 指定 |
| 无图形界面 / 无 GNOME 代理配置 | 需 `export http_proxy=...` 或改 prefs 直连 |

**绝大多数情况直接跑即可**，例如：

```bash
# 系统已配好代理，直接跑
./capture.sh "https://www.youtube.com/watch?v=_n4SRDYkhqs"

# 临时想换个端口 / 直连
# export http_proxy=socks5://127.0.0.1:1080  # 可选
./capture.sh "https://www.youtube.com/watch?v=_n4SRDYkhqs"
```

## 关键点

- **同一 patched Firefox 两阶段**，非两套 binary  
- **Profile 复用**：Phase 1 写 cookie → Phase 2 读 cookie → cf_clearance 复用  
- **Sandbox**：Phase 1 开（过 CF），Phase 2 关（StreamDumper 写盘）  
- **监控**：lib-monitor.sh（BiDi 5s 轮询 ended/stalled/paused/next_episode），崩溃自动恢复 ≤3 次（seek 关键帧续接）  
- **切集检测**：`location.href` 变 + `video.currentSrc` 变 + `readyState≥2` 三重确认  
- **Codec 自动**：H.264/HEVC/AV1 dispatch，AV1 自动包 IVF  
- **音频同步**：AAC 双轨同步 dump，合流自动对齐

## 排查

```bash
# 进度、关键帧、切集 URL
cat ~/Videos/firefox抓取/{host}-*.sidecar.json

# Firefox + monitor 日志
cat ~/Videos/firefox抓取/{host}-*.log
```

| 现象 | 调整 |
|---|---|
| Phase 1 超时 | 改 `PHASE1_TIMEOUT=1800`（秒） |
| Phase 2 无增长兜底结束 | 改 `MONITOR_STALL_LIMIT=36`（轮询次数，默认 5s×36=180s） |
| 无 dump | 确认视频真在播（`<video>` 未被广告遮挡 / 暂停） |
