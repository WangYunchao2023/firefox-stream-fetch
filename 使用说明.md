# firefox-stream-fetch v2.0.0 使用说明

## 一句话

用修改版 Firefox 打开视频网站 → 过人机验证 → 自动抓取解密后的 H.264 码流 → 转成 .mp4。

## 一键抓取

```bash
cd /home/wangyc/Documents/软件类工作/firefox-stream-fetch

# 方式 A：空白页启动，在 Firefox 里手动输入 URL（推荐）
./scripts/capture-generic.sh

# 方式 B：直接指定 URL
./scripts/capture-generic.sh https://www.yfsp.tv/play/zImbBGABDR2

# 方式 C：指定保存目录 + 自动连集
./scripts/capture-generic.sh --output /data/videos/ --auto-next
```

## 流程

```
启动脚本 → Phase 1 (sandbox 正常)
  → 浏览器打开（输入 URL → CF 验证 → 视频播放）
  → 脚本自动检测到"播放开始"，关闭 Firefox（存 session）
  → Phase 2 (sandbox 关闭 + session restore)
  → 视频恢复播放 → StreamDumper 写 .h264
  → 120 秒无增长 → ffmpeg 转 .mp4 → 完成
```

## 输出位置

默认：`~/Videos/firefox抓取/`（可用 `--output` 修改）

```
文件：{域名}-{时间}.h264   ← 原始裸流
       {域名}-{时间}.mp4    ← 最终产物（可播放）
```

## 常用命令行参数

| 参数 | 作用 | 默认 |
|---|---|---|
| `--output DIR` | 保存目录 | `~/Videos/firefox抓取/` |
| `--auto-next` | 开启连集 | 关 |
| `--framerate N` | mp4 帧率 | 15 |
| `--no-cleanup` | 保留 .h264 不删 | 删 |

## 注意

- **CF 验证**：第一次需要手动过人机验证，cookie 复用好几个小时
- **Firefox 窗口**：启动后如果看不到窗口，切到 :1 桌面或 Alt+Tab 找
- **不要动这些**：`general.useragent.override`、`security.sandbox.*.level=0`（脚本已自动处理）
- **只对 L3 解密有效**（桌面 Linux 默认）
