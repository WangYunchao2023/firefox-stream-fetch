# firefox-stream-fetch v3.0 使用说明

## 一句话

用同一 patched Firefox 两阶段抓取：Phase 1 打开页面、过 CF/登录 → Phase 2 同 profile 从头重放、StreamDumper 抓解密后的 H.264/HEVC/AV1 + AAC → 自动合流 .mp4。

---

## 快速开始

```bash
cd /home/wangyc/Documents/软件类工作/firefox-stream-fetch/scripts

# 1. 明文直连站（最快验证，无 CF、无代理）
./capture.sh "https://www.olevod.com/player/vod/1-82695-1.html"

# 2. 需要代理的站（YouTube / 国外）
export http_proxy=socks5://127.0.0.1:1080
export https_proxy=socks5://127.0.0.1:1080
./capture.sh "https://www.youtube.com/watch?v=_n4SRDYkhqs"

# 3. 需要 CF 验证的站（yfsp 等）
./capture.sh "https://www.yfsp.tv/watch?v=ztgsSWh5mPZEhhazLjYUG6"
# → Phase 1 你手动点过验证 → 自动切 Phase 2 抓流

# 4. 完全手动（不传 URL，在 Firefox 地址栏输）
./capture.sh

# 5. 保留中间文件 / 自定义输出目录
./capture.sh "https://..." --keep-h264 --output /data/videos

# 6. 自动连集（每集独立 mp4：host-01.mp4, host-02.mp4...）
./capture.sh "https://www.yfsp.tv/watch?v=xxx" --auto-next
# → Phase 1 你手动过验证 → Phase 2 抓第一集 → 自动检测下一集 → host-02.mp4 → ...
```

---

## 流程图解

```
Phase 1（用户交互，sandbox 开启，真实 UA）
  ├─ 启动 patched Firefox（同 profile）
  ├─ 打开 URL（或 about:newtab 让你手输）
  ├─ 遇到 CF：你在浏览器里手动过验证
  ├─ 窗口标题从 "Just a moment..." 变成视频标题
  ├─ 脚本自动 SIGTERM Firefox（保存 session + cookie）
  └─ 检测到 cf_clearance cookie → 进入 Phase 2

Phase 2（自动抓取，sandbox 关闭）
  ├─ 同 profile 重启 Firefox
  ├─ 同 URL 从头加载（保证 codec init 完整）
  ├─ StreamDumper 在 MediaFormatReader hook 写 .h264/.aac
  ├─ lib-monitor 每 5s BiDi 轮询 <video>.ended / stalled / paused
  ├─ 视频自然结束 → ffmpeg -c copy 合流 .mp4
  └─ 清理中间文件（除非 --keep-h264）
```

## 常用参数

| 参数 | 作用 | 默认 |
|---|---|---|
| `--output DIR` | 保存目录 | `~/Videos/firefox抓取/` |
| `--keep-h264` | 保留 .h264/.aac 中间文件 | 合流后删 |
| `--skip-phase1` | 跳过 Phase 1（profile 已有 cf_clearance） | 否 |
| `--profile PATH` | 自定义 profile 路径 | `/tmp/firefox-stream-{host}` |
| `--auto-next` | **开启自动连集**（每集独立 mp4：host-01.mp4, host-02.mp4...） | 否 |
| `--help` | 帮助 | - |

## 代理设置（统一）

脚本**读取环境变量**，不写死：

```bash
export http_proxy=socks5://127.0.0.1:1080
export https_proxy=socks5://127.0.0.1:1080
# 或
export all_proxy=socks5://127.0.0.1:1080
```

- Phase 1：显式 unset 代理（避免 CF 抓 proxy 痕迹）
- Phase 2：自动恢复环境变量，Firefox 走系统代理
- 未设任何 `*_proxy` → 直联

## 关键点

- **两阶段用同一 patched Firefox**，不是两套 binary
- **Profile 复用**：Phase 1 写 cookie → Phase 2 同 profile 读 cookie → cf_clearance 复用
- **Sandbox**：Phase 1 开（过 CF），Phase 2 关（让 StreamDumper 写盘）
- **监控**：lib-monitor.sh（BiDi 5s 轮询 `video.ended / stalled / paused / no-video`），崩溃自动恢复 ≤ 3 次（seek 到关键帧续接）
- **Codec 自动**：H.264/HEVC/AV1 自动 dispatch，AV1 自动包 IVF 容器
- **音频同步**：AAC 双轨同步 dump，合流时自动对齐

## 排查

```bash
# 看 monitor 日志
cat ~/Videos/firefox抓取/{host}-{timestamp}.log

# 看 Firefox stdout/stderr
cat ~/Videos/firefox抓取/{host}-{timestamp}.log

# 看 sidecar 状态（进度、关键帧、中断历史）
cat ~/Videos/firefox抓取/{host}-{timestamp}.sidecar.json
```

常见情况：
- **Phase 1 30 分钟超时**：改脚本里 `PHASE1_TIMEOUT=1800`
- **Phase 2 120s 无增长兜底结束**：长视频可改 `MONITOR_STALL_LIMIT`（默认 36 次 × 5s = 180s）
- **无 dump**：确认视频真在播放（`<video>` 没被广告遮挡 / 暂停）
